# 打字练习体验优化设计

## 目标

在不改技术栈、不加后端、不做大规模重构的前提下，优化当前打字练习应用的练习体验、统计反馈和视觉可读性。重点让用户更顺手地进入下一轮练习，更清楚地看到近期表现，并让桌面与移动端界面更稳定。

## 范围

本轮包含三类优化：

- 练习体验：增加键盘快捷键、连续正确反馈、完成后更快进入下一段。
- 统计能力：增加历史成绩摘要，包括最佳 WPM、平均 WPM、平均准确率和总练习次数。
- 视觉细节：强化练习文本主区域，压缩统计信息噪音，优化历史记录和移动端排版。

本轮不包含登录、后端、云同步、复杂图表、状态库迁移和专项代码重构。

## 练习体验设计

用户在练习页可以继续用鼠标操作，也可以使用键盘完成常见控制：

- `Esc`：重练当前段落。
- `Ctrl + Enter` 或 `Command + Enter`：换下一段。

页面在完成、重练、切换模式或下一段后都会尽量让输入框重新获得焦点。连续正确输入时，界面展示轻量的连击状态；输入错误时连击清零，并沿用已有的错误提示音开关。

## 统计反馈设计

基于已有 localStorage 历史数据，不新增数据来源。历史区上方增加一组摘要：

- 练习次数：当前保存的最近记录数量。
- 最佳 WPM：最近记录里的最高速度。
- 平均 WPM：最近记录的平均速度。
- 平均准确率：最近记录的平均准确率。

空历史时展示清晰的初始状态，不让用户误以为统计损坏。

## 视觉设计

保留当前“练字舱”的深色专注风格，但调整信息层级：

- 练习文本区更突出，当前字符和错误字符更容易辨认。
- 统计面板保持紧凑，避免数字卡片在移动端显得拥挤。
- 历史记录改成更易扫描的成绩列表，并加入摘要区。
- 移动端减少大块间距，保证按钮、统计和文本不溢出。

## 边界与错误处理

- 快捷键只在页面内生效，不阻止用户在输入框中正常输入文字。
- 没有历史数据时，摘要显示合理默认值。
- localStorage 失败时，练习功能继续可用。
- 计时结束或段落完成后，重复触发完成逻辑不会重复写入历史。

## 验证

- 运行 `npm run build` 确认 TypeScript 和 Vite 构建通过。
- 启动本地开发服务器，验证桌面视口和移动视口。
- 手动验证快捷键、下一段、重练、历史摘要、完成状态和移动端排版。
